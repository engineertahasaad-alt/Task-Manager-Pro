ISSUE 1 - Task Reassignment Workflow
====================================

Current bug:

1. User requests task reassignment.
2. Manager receives the notification too late.
3. Manager rejects the request.
4. User's task screen becomes stuck.
5. Only "Request Reassignment" button remains.
6. Normal task actions disappear:

   * Mark as Done
   * Start
   * Pause
   * Resume
   * Continue
   * Any action allowed before reassignment request.

Expected behavior:

A reassignment request is only a temporary request.

If manager rejects:

* Restore the task to exactly the previous active state.
* Restore every action button according to task status.
* User should continue working normally.
* No UI dead-end.

If manager approves:

* Transfer ownership correctly.
* Update permissions.
* Refresh task state for both users.
* Remove old owner's action buttons.
* New owner receives correct actions.

Verify every transition.
