# Taskaya - Architecture Review, Refactoring & Full System Validation

Please perform a complete architecture review of the entire application (Backend, Database, Web App, Mobile App) before implementing any changes.

The objective is to deliver a production-ready system, not a prototype, partial implementation, workaround, or disconnected MVP.

---

## 1. Multi-Group Membership & Roles

The system must support:

* A single user belonging to multiple groups simultaneously.
* A single user being a Manager in multiple groups.
* A single user being a Team Member in multiple groups.
* A single user being a Manager in one group and a Team Member in another group at the same time.

Permissions must be determined per group, not per user.

All screens, APIs, dashboards, reports, notifications, filters, permissions, and workflows must respect the user's role within the currently selected group.

If the current architecture does not support this correctly, redesign and refactor it properly.

---

## 2. Database & Architecture Refactoring

I have no issue with database refactoring if required.

Do not implement temporary fixes or workarounds.

If necessary, redesign the database to properly support:

* Users
* Groups
* Group Memberships
* Group Roles
* Tasks
* Task Assignees
* Task Delegations
* Notifications
* Audit Logs

Use scalable relational modeling.

All permissions must be group-based.

---

## 3. Single Shared Backend & Database

The Web App and Mobile App must use:

* The same backend
* The same APIs
* The same database

A user who registers from Web must be able to log in immediately from Mobile using the same credentials.

A user who registers from Mobile must be able to log in immediately from Web using the same credentials.

No separate databases.

No duplicated user records.

No duplicated workspaces.

---

## 4. Full Synchronization Between Web & Mobile

Verify complete synchronization of:

* Users
* Groups
* Roles
* Tasks
* Task Assignments
* Task Statuses
* Comments
* Notifications
* Dashboards
* Reports
* Activity Logs

Any change performed from Web must appear on Mobile.

Any change performed from Mobile must appear on Web.

Review all synchronization logic and fix inconsistencies.

## 5. Multi-Assignee Tasks

Allow tasks to be assigned to multiple users simultaneously.

Requirements:

* All assigned users can view the task.
* All assigned users receive notifications.
* Progress tracking remains accurate.
* Task history remains intact.
* UI clearly displays all assigned members.
* Mobile and Web support the same functionality.

---

## 6. Cross-Group Task Delegation

Support the following scenario:

Manager A receives a task from their manager in Group A.

Manager A must be able to delegate or distribute that task to members of another group they manage (Group B).

Requirements:

* Preserve original task history.
* Preserve original creator.
* Preserve timestamps.
* Preserve audit logs.
* Preserve traceability.
* Create a clear parent-child relationship between original and delegated tasks.

Managers must be able to track:

* Original task
* Delegated tasks
* Progress of delegated tasks
* Completion status

## 8. Critical Bug - Team Code Registration

A major issue currently exists.

Current behavior:

When a user registers using a Team Code / Group Code:

* A new workspace is created.
* The user becomes Owner.
* The user is NOT added to the intended team.

This is incorrect.

Required behavior:

When a valid Team Code is entered:

* User joins the existing team.
* No new workspace is created.
* User receives the intended role.
* Membership record is created correctly.
* Existing team data becomes visible immediately.

Review and fix:

* Registration flow
* Invitation flow
* Team Code validation
* Membership creation
* Role assignment
* Workspace creation logic
* Authentication logic
* Mobile onboarding
* Web onboarding

Do not only fix the symptom.

Perform full root-cause analysis.
